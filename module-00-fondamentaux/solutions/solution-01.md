# Solutions Exercice 01 : Conversions Numériques

---

## Partie 1 : Binaire → Décimal

1. `1010` = 8 + 2 = **10**
2. `11111111` = 128+64+32+16+8+4+2+1 = **255**
3. `10000000` = 128 = **128**
4. `01010101` = 64+16+4+1 = **85**
5. `11001010` = 128+64+8+2 = **202**
6. `00001111` = 8+4+2+1 = **15**
7. `10101010` = 128+32+8+2 = **170**
8. `11110000` = 128+64+32+16 = **240**

### Méthode rapide

```
Position:  7   6   5   4   3   2   1   0
Valeur:  128  64  32  16   8   4   2   1

Pour 11001010:
         1   1   0   0   1   0   1   0
       128 +64 + 0 + 0 + 8 + 0 + 2 + 0 = 202
```

## Partie 2 : Décimal → Binaire

1. `42` = **00101010**
   ```
   42 ÷ 2 = 21 reste 0
   21 ÷ 2 = 10 reste 1
   10 ÷ 2 = 5  reste 0
   5  ÷ 2 = 2  reste 1
   2  ÷ 2 = 1  reste 0
   1  ÷ 2 = 0  reste 1
   Lire de bas en haut: 101010 → 00101010
   ```

2. `255` = **11111111**
3. `128` = **10000000**
4. `100` = **01100100**
5. `17` = **00010001**
6. `200` = **11001000**
7. `65` = **01000001**
8. `1` = **00000001**

### Méthode alternative (soustraction)

Pour 100:
- 100 ≥ 128 ? Non → 0
- 100 ≥ 64 ? Oui → 1, reste 36
- 36 ≥ 32 ? Oui → 1, reste 4
- 4 ≥ 16 ? Non → 0
- 4 ≥ 8 ? Non → 0
- 4 ≥ 4 ? Oui → 1, reste 0
- 0 ≥ 2 ? Non → 0
- 0 ≥ 1 ? Non → 0

Résultat: 01100100

## Partie 3 : Hexadécimal → Décimal

1. `0x10` = 1×16 + 0 = **16**
2. `0xFF` = 15×16 + 15 = 240 + 15 = **255**
3. `0xAB` = 10×16 + 11 = 160 + 11 = **171**
4. `0x7F` = 7×16 + 15 = 112 + 15 = **127**
5. `0xDEAD` = 13×4096 + 14×256 + 10×16 + 13 = 53248 + 3584 + 160 + 13 = **57005**
6. `0xCAFE` = 12×4096 + 10×256 + 15×16 + 14 = 49152 + 2560 + 240 + 14 = **51966**
7. `0x1337` = 1×4096 + 3×256 + 3×16 + 7 = 4096 + 768 + 48 + 7 = **4919**
8. `0x0A` = 0×16 + 10 = **10**

---

## Partie 4 : Décimal → Hexadécimal

1. `16` = **0x10**
2. `255` = **0xFF**
3. `4096` = **0x1000**
4. `256` = **0x100**
5. `1000`:
   ```
   1000 ÷ 16 = 62 reste 8
   62 ÷ 16 = 3 reste 14 (E)
   3 ÷ 16 = 0 reste 3
   → 0x3E8
   ```
6. `127` = **0x7F**
7. `48879` = **0xBEEF** (48879 = 11×4096 + 14×256 + 14×16 + 15)
8. `3735928559` = **0xDEADBEEF**

## Partie 5 : Binaire ↔ Hexadécimal

### Binaire → Hexadécimal

1. `11111111` → `1111 1111` → F F → **0xFF**
2. `10101010` → `1010 1010` → A A → **0xAA**
3. `11110000 00001111` → `1111 0000 0000 1111` → F 0 0 F → **0xF00F**
4. `01001000 01100101` → `0100 1000 0110 0101` → 4 8 6 5 → **0x4865**
   (Note: 0x48 = 'H', 0x65 = 'e' en ASCII !)

### Hexadécimal → Binaire

1. `0xDE` → D=1101, E=1110 → **11011110**
2. `0xAD` → A=1010, D=1101 → **10101101**
3. `0xBEEF` → B=1011, E=1110, E=1110, F=1111 → **1011111011101111**
4. `0x4142` → 4=0100, 1=0001, 4=0100, 2=0010 → **0100000101000010**
   (Note: 0x41 = 'A', 0x42 = 'B' en ASCII !)

## Partie 6 : Exercices Pratiques

### Calcul de tailles

1. 4 bits → 2^4 = **16 valeurs** (0 à 15)
2. 8 bits → 2^8 = **256 valeurs** (0 à 255)
3. 16 bits → 2^16 = **65536 valeurs** (0 à 65535)
4. Pour représenter 1000 : 2^9 = 512, 2^10 = 1024 → **10 bits minimum**

### Manipulation de bytes

1. 2048 bytes = 2048/1024 = **2 KB**
2. 32 bits → 2^32 bytes = **4 294 967 296 bytes = 4 GB**
3. Entier non signé 16 bits max = 2^16 - 1 = **65535**

### Opérations binaires

1. `1010 AND 1100`:
   ```
       1010
   AND 1100
       ────
       1000
   ```
   Résultat: **1000** (= 8 en décimal)

2. `1010 OR 1100`:
   ```
       1010
   OR  1100
       ────
       1110
   ```
   Résultat: **1110** (= 14 en décimal)

3. `1010 XOR 1100`:
   ```
       1010
   XOR 1100
       ────
       0110
   ```
   Résultat: **0110** (= 6 en décimal)

4. `NOT 1010` (sur 4 bits):
   ```
   NOT 1010
       ────
       0101
   ```
   Résultat: **0101** (= 5 en décimal)
